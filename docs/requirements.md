# 阶段 1 需求口径

本文档固化当前本地仓库的 MVP 口径，作为后续协作、接口对齐和任务拆分的基础。当前口径以代码实现为准，主要参考后端 Controller、权限服务、数据库迁移脚本、前端路由和 README。

## MVP 目标

新员工“一人一画像”培养管理平台用于支撑新员工培养过程中的人员管理、学习记录、闯关任务、考试评价、综合评分和权限分层管理。当前 MVP 的目标是形成可本地运行、可演示核心业务闭环、可继续多人协作开发的版本。

当前 MVP 需要回答三个问题：

- 培训管理方能维护人员、课程、任务、考试和评价规则。
- 新员工能查看自己的培养过程，完成签到、任务提交和考试。
- 导师、站点负责人、培训管理员、管理员和超级管理员能按角色权限处理各自范围内的数据。

## 目标用户

| 角色 | 当前定位 |
| --- | --- |
| `EMPLOYEE` | 新员工，查看本人进度，完成课程签到、任务提交、考试作答，查看已发布评价结果。 |
| `MENTOR` | 导师，查看所带员工，提交导师月度评价。 |
| `STATION_MANAGER` | 站点负责人，查看绑定服务站员工，提交站点评价。 |
| `TRAINING_ADMIN` | 培训管理员，管理课程、签到、任务、考试、评价，提交培训方评价。 |
| `ADMIN` | 管理员，维护员工台账、基础数据、员工账号、审计查询和大部分业务配置。 |
| `SUPER_ADMIN` | 超级管理员，拥有管理员能力，并可管理非员工角色账号。 |

## 核心业务闭环

1. 超级管理员或管理员准备基础数据，包括培养批次、服务站、导师和员工台账。
2. 管理端创建员工账号。员工账号默认需要首次改密，导入员工时账号默认停用。
3. 培训管理员创建课程和场次，将员工安排到课程场次。
4. 员工使用签到码在签到时间窗口内完成课程签到；培训管理员可补录或导入签到记录。
5. 培训管理员创建闯关任务并按人员、批次或服务站分配。
6. 员工提交任务成果和附件；有审核权限的角色进行通过、退回和评分。
7. 考试管理员维护题库、试卷和考试计划，将考试分配给员工并发布。
8. 员工在开放时间内开始考试、自动保存答案、提交答卷；客观题自动评分，主观题由考试管理员阅卷后发布结果。
9. 评价管理员维护评分方案，系统汇总考试、任务、导师评价、站点评价、培训评价和加扣分。
10. 评价管理员生成并发布月度或季度汇总；管理员可在需要时重开已发布月度汇总。
11. 用户按角色和数据范围查看进度概览、人员信息、评价结果和审计记录。

## 当前已支持范围

### 登录与安全

- 支持用户名密码登录，登录成功后返回 JWT 和当前用户信息。
- 支持 `/api/v1/auth/me` 获取当前登录用户。
- 支持首次登录强制修改密码。
- 账号停用、角色调整和密码重置会递增 `security_version`，旧令牌失效。
- 前端路由会拦截未登录用户，并在 `mustChangePassword=true` 时强制进入个人设置页。

### 人员台账与人员目录

- 支持员工分页查询、详情查询、创建、更新和批量绑定导师。
- 支持员工 Excel 模板下载和整批导入；任一行校验失败时整批不写入。
- 支持人员目录多条件查询和 Excel 导出。
- 员工台账详情、课程、签到、任务、评价、考试等员工关联查询受数据范围约束：本人、导师所带员工、站点范围或全量。

### 基础数据

- 支持培养批次和服务站查询。
- 支持管理员创建培养批次和服务站。
- 支持查询启用状态的导师账号。

### 课程与签到

- 支持课程创建和查询。
- 支持课程场次创建和查询。
- 支持将员工安排到课程场次。
- 支持员工在签到窗口内使用 6 位签到码签到。
- 支持管理员补录签到和导入签到记录。
- 支持按员工范围查看签到记录。

### 闯关任务

- 支持创建任务。
- 支持按员工、批次或服务站分配任务。
- 支持员工提交文字说明和最多 5 个附件。
- 支持任务提交历史查看。
- 支持审核通过、退回和评分。
- 支持任务附件下载，下载权限按任务所属员工的数据范围控制。

### 综合评价

- 支持批次评分方案草稿、发布、复制为新草稿、删除。
- 支持考试、任务、导师、站点、培训五类评分项启停和权重配置。
- 支持月度评价明细、手工评分、评分项覆盖、加扣分、月度汇总、季度汇总。
- 支持发布汇总，缺失评分项时要求豁免原因和人工核定总分。
- 支持管理员重开已发布月度汇总。

### 考试中心

- 支持单选、多选、判断题题库维护。
- 支持题库模板下载和 Excel 导入。
- 支持手动组卷和随机组卷，试卷总分必须为 100 分。
- 支持考试计划创建、发布、补充分配和次数限制。
- 支持员工开考、保存答案、防作弊事件记录和提交答卷。
- 支持阅卷队列、主观题评分和考试结果发布。
- 支持查看已发布考试结果。

### 账号权限与审计

- 支持员工账号列表、启停和重置密码。
- 超级管理员支持创建和管理导师、站点负责人、培训管理员、管理员账号。
- 支持站点负责人绑定服务站范围。
- 支持审计日志查询，覆盖登录、创建、更新、导入、发布、审核等关键操作。

### 进度概览

- 支持按当前用户数据范围展示人员数、任务总数、已完成任务数、平均评分、签到数和任务评分分布。

## 当前不支持范围

当前 MVP 明确不覆盖以下能力：

- 生产级 SSO、LDAP、企业微信或钉钉统一认证。
- 短信、邮件、站内信或待办通知。
- 外部 HR、OA、LMS、考试系统或数据仓库同步。
- 多级审批流、电子签章和复杂流程编排。
- 原生移动 App、小程序或离线端。
- 真实实时音视频监考、摄像头抓拍、人脸识别和 AI 作弊判定。
- 生产部署拓扑、高可用、备份恢复、容量规划和 SLA 规范。
- 细粒度字段级脱敏、租户隔离、组织树和动态权限配置。
- 完整 OpenAPI/Swagger 自动生成合同。

## 验收口径

- 新协作者能从本文档理解当前 MVP 做什么、不做什么。
- 涉及角色、权限、接口和任务拆分时，以 `docs/api-contract.md`、`docs/permissions-matrix.md` 和 `docs/task-board.md` 为协作入口。
- 如果后续代码改动改变业务边界、接口形态或权限行为，必须同步更新对应文档。
