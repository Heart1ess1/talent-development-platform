## 使用规则

- 新任务进入 `Backlog`，需求明确且可开发时改为 `Ready`。
- 开始开发时改为 `In Progress`，遇到外部依赖或无法推进的问题改为 `Blocked`。
- 合并或验收完成后改为 `Done`，并在备注中记录关键 PR 或验证方式。
- 涉及接口、权限或 MVP 边界的任务完成时，必须同步更新阶段 1 文档。
